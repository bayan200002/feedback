# Lab Task – Gathering and Integrating Feedback

## Part 1 – Multiple Choice Questions

1. Why is user feedback important in documentation?

Answer:
b) It helps improve documentation and products

2. Which feedback channel allows users to report issues on a specific documentation page?

Answer:
c) Page-level feedback

3. What is the main purpose of triage?

Answer:
b) Sorting and prioritizing feedback

4. Which of the following is a limitation of document sentiment tools?

Answer:
a) They require many responses

5. Which feedback channel provides structured and measurable feedback?

Answer:
a) User surveys

| Field | Example |
|---------|---------|
| Issue Title | Broken Password Reset Instructions |
| Document URL | docs.bankapp.com/reset-password |
| Problem Description | Step 3 does not match the current app screen |

### Example Issue

Issue Title:
Broken Password Reset Instructions

Document URL:
docs.bankapp.com/reset-password

Problem Description:
Step 3 does not match the current application screen and causes confusion for users.

Suggested Solution:
Update the screenshot and rewrite Step 3 to match the latest version of the application.

---

## Conclusion

User feedback is essential for improving documentation quality. Feedback can be collected through surveys, support issues, page-level feedback, and user councils. Triage helps prioritize issues so teams can focus on the most important problems first.
